Future Systems

Ideas for later — not the current promise.

Earlier versions of this site described token, marketplace, and wallet systems as if they were already central. The active game direction is now game-first: polish the playable build before promising extra systems.

Play Current BuildRead Source Policy
Current Rule

Gameplay clarity comes first.

1

Current build first

The website should describe what the active build actually does before describing future economy ideas.

2

No hidden promises

Tokens, marketplaces, skins, or wallet features should be described as future concepts unless they are implemented and tested.

3

No pay-to-win

Any future system must preserve fair gameplay. Players should never need payment or wallet knowledge to enjoy the core game.

Future Concept

Possible economy layer, only after the game is strong.

A future economy layer could support cosmetic ownership, optional collectibles, or community rewards. That layer should be invisible to players who only want to play.

The current cleanup intentionally removes hard claims about active token rewards, exact token supply, fees, chain choice, and wallet behavior from the main promise. Those details should return only after they are implemented, audited, and documented.

Build Order

First make the game trustworthy.

The website should make players trust the game, the history, and the direction. Extra systems can come later.
